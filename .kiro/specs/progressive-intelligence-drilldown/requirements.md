# Requirements Document

## Introduction

Transform the static "Investigative Questions to Pursue" section in the entity intelligence panel (`investigator.html`) into an interactive progressive drill-down system. Currently, when an investigator clicks a node on the knowledge graph, the entity intelligence panel displays AI-generated investigative questions as static text (rendered by `_loadAIIntelligence` from the `/patterns` API and by `_generateEntityQuestions` client-side). This feature makes each question clickable with three progressive levels of AI-generated answers, plus a contextual follow-up question bar scoped to the entity's graph neighborhood. All changes are additive — existing working code in `investigator_ai_engine.py`, `patterns.py`, `chat_service.py`, and `investigator.html` is extended, not replaced.

## Glossary

- **Intelligence_Panel**: The entity drill-down panel in `investigator.html` that appears when an investigator clicks a node on the knowledge graph, displaying entity profile, AI analysis, investigative questions, and key insights.
- **Investigative_Question**: A question generated by the AI engine (via the `/patterns` API `investigative_questions` field) or by the client-side `_generateEntityQuestions` function, scoped to an entity's graph neighborhood and document references.
- **Level_1_Answer**: A one-line quick summary answer for an Investigative_Question, generated alongside the question during the initial AI analysis call with zero additional latency.
- **Level_2_Answer**: A 2–3 paragraph analytical brief with document citations for an Investigative_Question, fetched on-demand via semantic search and graph context when the investigator clicks "Expand".
- **Level_3_Answer**: A full structured intelligence report for an Investigative_Question containing executive summary, evidence analysis, source citations with document links, confidence assessment, and recommended next steps, displayed in a modal or slide-out panel.
- **Follow_Up_Bar**: A contextual text input scoped to the current entity and its graph neighborhood, allowing the investigator to type a custom question and receive answers in the same three-level progressive format.
- **Patterns_API**: The existing `POST /case-files/{id}/patterns` endpoint in `patterns.py` that generates entity intelligence analysis via Bedrock, returning `investigative_questions`, `connection_intelligence`, `recommended_actions`, and other fields.
- **Question_Answer_API**: A new `POST /case-files/{id}/question-answer` endpoint that generates Level 2 and Level 3 answers on-demand using semantic search, graph context, and Bedrock AI synthesis.
- **Graph_Neighborhood**: The set of entities and relationships within 1–2 hops of the current entity in the Neptune knowledge graph.

## Requirements

### Requirement 1: Level 1 Quick Answers Generated with Initial Analysis

**User Story:** As an investigator, I want each investigative question to include a one-line quick answer visible on first click, so that I can rapidly triage which questions deserve deeper investigation without waiting for additional API calls.

#### Acceptance Criteria

1. WHEN the Patterns_API generates `investigative_questions` for an entity, THE Patterns_API SHALL return each question as an object containing a `question` text field and a `quick_answer` text field of one sentence or fewer.
2. THE Patterns_API SHALL generate Level_1_Answers within the same Bedrock invocation that generates the investigative questions, adding zero additional API calls or latency.
3. WHEN the Intelligence_Panel renders an Investigative_Question, THE Intelligence_Panel SHALL display the question text as a clickable element that reveals the Level_1_Answer on first click.
4. THE Level_1_Answer SHALL be a single sentence of 150 characters or fewer that directly addresses the question using the entity's graph neighborhood and document context.
5. IF the Patterns_API fails to generate a Level_1_Answer for a question, THEN THE Intelligence_Panel SHALL display the question as clickable with a fallback message "Quick answer unavailable — click Expand for full analysis."

### Requirement 2: Level 2 Analytical Brief On-Demand

**User Story:** As an investigator, I want to expand any investigative question into a 2–3 paragraph analytical brief with document citations, so that I can evaluate the evidentiary basis for each question before committing to a full intelligence report.

#### Acceptance Criteria

1. WHEN the investigator clicks "Expand" on a question that has a visible Level_1_Answer, THE Intelligence_Panel SHALL call the Question_Answer_API to fetch a Level_2_Answer.
2. THE Question_Answer_API SHALL accept a case file ID, entity name, question text, and answer level parameter, and return a Level_2_Answer.
3. THE Question_Answer_API SHALL use semantic search over case documents and graph neighborhood context to generate the Level_2_Answer via Bedrock.
4. THE Level_2_Answer SHALL contain 2–3 paragraphs of analytical text and a list of document citations, where each citation includes a document name and relevance indicator.
5. WHILE the Level_2_Answer is loading, THE Intelligence_Panel SHALL display a loading indicator with the text "Generating analytical brief..." within the question card.
6. THE Question_Answer_API SHALL return the Level_2_Answer within 5 seconds under normal operating conditions.
7. IF the Question_Answer_API fails to generate a Level_2_Answer, THEN THE Intelligence_Panel SHALL display an error message "Analysis unavailable" with a "Retry" button within the question card.

### Requirement 3: Level 3 Intelligence Report

**User Story:** As an investigator, I want to generate a full structured intelligence report for any investigative question, so that I can document findings with executive summaries, evidence analysis, confidence assessments, and recommended next steps for case files.

#### Acceptance Criteria

1. WHEN the investigator clicks "Full Report" on a question that has a visible Level_2_Answer, THE Intelligence_Panel SHALL call the Question_Answer_API with level 3 to fetch a Level_3_Answer.
2. THE Level_3_Answer SHALL contain the following sections: executive summary, evidence analysis, source citations with document links, confidence assessment (high/medium/low with justification), and recommended next steps.
3. THE Intelligence_Panel SHALL display the Level_3_Answer in a modal overlay or dedicated slide-out panel, separate from the inline question card.
4. WHILE the Level_3_Answer is loading, THE Intelligence_Panel SHALL display a loading indicator with the text "Generating intelligence report..." in the modal or panel.
5. THE Question_Answer_API SHALL return the Level_3_Answer within 10 seconds under normal operating conditions.
6. THE Level_3_Answer SHALL use the same senior federal investigative analyst persona and formatting style as the existing AI analysis generated by the Patterns_API.
7. IF the Question_Answer_API fails to generate a Level_3_Answer, THEN THE Intelligence_Panel SHALL display an error message in the modal with a "Retry" button.

### Requirement 4: Contextual Follow-Up Question Bar

**User Story:** As an investigator, I want to type custom follow-up questions scoped to the current entity and its graph neighborhood, so that I can pursue lines of inquiry beyond the AI-generated questions without leaving the entity intelligence panel.

#### Acceptance Criteria

1. THE Intelligence_Panel SHALL display a Follow_Up_Bar input field below the investigative questions section when an entity is selected.
2. WHEN the investigator submits a question via the Follow_Up_Bar, THE Intelligence_Panel SHALL call the Question_Answer_API with the custom question, entity name, and case file ID.
3. THE Question_Answer_API SHALL scope the follow-up answer to the current entity's Graph_Neighborhood and case documents, using semantic search and graph context.
4. THE Intelligence_Panel SHALL display the follow-up answer in the same three-level progressive format as AI-generated Investigative_Questions, with Level_1_Answer visible immediately and "Expand" and "Full Report" buttons available.
5. WHILE the follow-up answer is loading, THE Intelligence_Panel SHALL display a loading indicator below the Follow_Up_Bar.
6. IF the investigator submits an empty question via the Follow_Up_Bar, THEN THE Intelligence_Panel SHALL display a validation message "Please enter a question" without calling the Question_Answer_API.
7. THE Follow_Up_Bar SHALL retain the investigator's question text after submission so the investigator can see what was asked.

### Requirement 5: Additive Backend Extension

**User Story:** As a developer, I want all backend changes to extend existing services additively, so that existing working functionality in the investigator analysis pipeline is preserved.

#### Acceptance Criteria

1. THE Patterns_API prompt in `_generate_entity_intelligence` SHALL be extended to include `quick_answer` fields alongside each `investigative_questions` entry, changing the response format from a list of strings to a list of objects.
2. THE Question_Answer_API SHALL be implemented as a new handler function in an existing or new Lambda module, routed through the existing API Gateway proxy configuration.
3. THE Question_Answer_API SHALL reuse the existing `SemanticSearchService` for document retrieval and the existing Neptune graph query patterns for graph context.
4. THE Question_Answer_API SHALL use the same Bedrock model ID and senior investigative analyst persona as the existing Patterns_API.
5. IF the Patterns_API response contains `investigative_questions` as a list of strings (legacy format), THEN THE Intelligence_Panel SHALL render the questions without Level_1_Answers and display "Click to analyze" as the initial state.

### Requirement 6: Progressive Disclosure UI Interaction

**User Story:** As an investigator, I want the question drill-down to follow a clear progressive disclosure pattern, so that I can control the depth of analysis and avoid information overload.

#### Acceptance Criteria

1. WHEN an Investigative_Question is first rendered, THE Intelligence_Panel SHALL display only the question text with a clickable indicator.
2. WHEN the investigator clicks a collapsed question, THE Intelligence_Panel SHALL reveal the Level_1_Answer and display an "Expand" button.
3. WHEN the investigator clicks "Expand", THE Intelligence_Panel SHALL fetch and display the Level_2_Answer below the Level_1_Answer and display a "Full Report" button.
4. WHEN the investigator clicks "Full Report", THE Intelligence_Panel SHALL fetch and display the Level_3_Answer in a modal or slide-out panel.
5. WHEN the investigator clicks a question that is already expanded, THE Intelligence_Panel SHALL collapse the answer back to the question-only state.
6. THE Intelligence_Panel SHALL cache fetched Level_2_Answers and Level_3_Answers in memory so that re-expanding a previously expanded question does not trigger a new API call.
