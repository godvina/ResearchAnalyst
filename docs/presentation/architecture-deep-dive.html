case_files
Core case management
  • case_id UUID PK
  • topic_name VARCHAR(255)
  • description TEXT
  • status created|ingesting|indexed|investigating|archived|error
  • parent_case_id UUID FK → self
  • s3_prefix VARCHAR(512)
  • neptune_subgraph_label VARCHAR(255)
  • document_count INT
  • entity_count INT
  • relationship_count INT
  • created_at TIMESTAMPTZ

documents
Ingested document content + embeddings
  • document_id UUID PK
  • case_file_id UUID FK → case_files
  • source_filename VARCHAR(512)
  • raw_text TEXT
  • sections JSONB
  • embedding vector(1024) — Titan v2
  • source_metadata JSONB
  • org_id UUID FK → organizations
  • matter_id UUID FK → matters
  • collection_id UUID FK → collections
  • security_label_override TEXT

entities
Extracted entities (Neptune primary)
  • entity_name VARCHAR — canonical form
  • entity_type person|location|date|artifact|organization|phone|email
  • case_file_id UUID FK
  • occurrence_count INT
  • confidence FLOAT 0.0–1.0

investigation_findings
Research notebook entries
  • finding_id UUID PK
  • case_file_id UUID FK
  • finding_type analysis|lead|note|hypothesis

  • metadata JSONB
  • created_at TIMESTAMPTZ

osint_research_cache
Cached OSINT research results
  • id UUID PK

🔗
Knowledge Graph Schema
Vertices, edges, and properties in Neptune Serverless

Vertices

🟢 Entity_{case_id}
  • canonical_name String — normalized form
  • entity_type person|location|date|artifact|civilization|theme|event|organization|phone|email
  • confidence Float 0.0–1.0
  • occurrence_count Integer

Edges

➡️ MENTIONED_IN (Entity → Document)
  • source_document String
➡️ CONNECTED_TO (Entity → Entity)
  • relationship_type co-occurrence|financial|geographic|temporal|causal|thematic
  • weight Float — connection strength
  • confidence Float 0.0–1.0
  • source_document String
  • case_file_id String (UUID)

Security Groups
Least-privilege network access
SGInboundPurpose
Lambda SG — All outbound allowed
Aurora SG TCP 5432 from VPC CIDR PostgreSQL access
Neptune SG TCP 8182 from VPC CIDR Gremlin HTTP API
AOSS VPCE SG TCP 443 from VPC CIDR + all Lambda SGs OpenSearch access
Bedrock VPCE SG TCP 443 from all Lambda SGs AI model access
SM VPCE SG TCP 443 from all Lambda SGs Secrets Manager
⚠️ Critical: Every Lambda SG must be added to every VPC endpoint SG it needs to reach. Missing rules cause silent "Connect timeout" failures.
