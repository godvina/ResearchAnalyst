Architecture Overview

Evidence seized during DEA drug bust operations is digitized and uploaded to Amazon S3 under the dea:// namespace prefix. Documents (ledgers, receipts, shipping manifests, ID cards) are processed through Amazon Textract for OCR and structured data extraction. Photographs (surveillance, mugshots, evidence photos) are analyzed by Amazon Rekognition for facial recognition, object detection, and scene analysis. All extracted text is then processed by Amazon Comprehend for named entity recognition (NER) identifying people, organizations, locations, dates, and monetary amounts. The resulting entities and relationships are loaded into Amazon Neptune in a dedicated dea: label prefix, sharing the same Neptune instance as DOJ case data but logically separated. Neptune's graph structure enables analysts to traverse connections between evidence items, suspects, locations, and cartel organizations. All processed text, entities, and metadata are then chunked and embedded into vectors using Amazon Titan Embeddings V2 and indexed in Amazon OpenSearch Serverless for semantic similarity search. Finally, an Amazon Bedrock Knowledge Base connects the OpenSearch vector store with Amazon Nova Pro, enabling analysts to ask natural language questions against the entire evidence corpus through retrieval-augmented generation (RAG).

Step 1 — Collection

📦 Physical Evidence Seizure

Field agents catalog and bag all evidence: paper files, ledgers, cell phones, hard drives, laptops, USB drives, cash, drugs, weapons, vehicles, and photographs. Each item receives a unique evidence tag and chain-of-custody form.

Step 2 — Digitization

📡 High-Resolution Scanning

Paper documents scanned at 600 DPI. Cell phones and hard drives forensically imaged using Cellebrite/EnCase. Photos digitized. All files uploaded to S3 with metadata tags (operation ID, evidence tag, date, location).

Step 3 — Document Processing

📄 Amazon Textract

Textract extracts text from scanned documents, identifies form fields in ledgers and receipts, and processes tables in financial records. Output stored as structured JSON in S3 alongside source files.

Step 4 — Image Analysis

📸 Amazon Rekognition

Rekognition performs facial detection and comparison against DEA watchlists, identifies objects (weapons, drugs, vehicles, currency), detects text in images, and analyzes scenes for location context.

Step 5 — Entity Extraction

🧠 Amazon Comprehend

Comprehend NER identifies people, organizations (cartels), locations, dates, and monetary amounts from all extracted text. Custom entity models trained on DEA-specific terminology for drug names, cartel aliases, and trafficking routes.

Step 6 — Knowledge Graph

🕸 Amazon Neptune

Entities become vertices (dea:Person, dea:Evidence, dea:Location, dea:Cartel). Relationships become edges (SEIZED_FROM, COMMUNICATES_WITH, MEMBER_OF, TRANSPORTED_TO). Gremlin queries enable multi-hop traversal to uncover hidden connections.

Step 7 — Vector Ingestion

🔍 Amazon OpenSearch Serverless

Extracted text from Textract, Comprehend entities, and Rekognition labels are chunked and embedded into 1,024-dimensional vectors using Amazon Titan Embeddings V2. Vectors are indexed in an OpenSearch Serverless (AOSS) collection for semantic similarity search across all evidence.

Step 8 — Knowledge Base & RAG

🤖 Amazon Bedrock Knowledge Base

A Bedrock Knowledge Base connects the OpenSearch vector store with Amazon Nova Pro as the foundation model. Analysts can ask natural language questions against the entire evidence corpus. The KB syncs automatically when new evidence is ingested, enabling real-time RAG-powered investigative queries.
